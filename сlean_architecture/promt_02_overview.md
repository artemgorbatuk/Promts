# Введение и архитектура проекта

### Требования

- Назначение: зафиксировать базовые принципы Clean Architecture, стек технологий и целевую структуру каталогов
- Документ должен быть кратким обзором без пошаговых инструкций реализации (они находятся в последующих promt файлах)
- Термины и названия слоев/папок должны соответствовать остальным promt файлам этой серии

Этот документ описывает ключевые принципы проекта: архитектуру, используемые технологии и структуру каталогов.

### Технологии и паттерны

- Архитектура: Clean Architecture с разделением на слои
- ORM: Entity Framework Core с PostgreSQL
- DI: Встроенная система внедрения зависимостей
- Паттерны: Repository, Unit Of Work

### Модели (общее)

- Модели слоя данных (Entities) — классы EF Core в `Datasource.Ef/Models`, отражают структуру таблиц и связей
- Опции репозиториев — классы `QueryOptions`/`CommandOptions` в `Repositories.Ef/Options`, задают параметры чтения (включая загрузку связей) и правила обновления связей в командах
- Модели сервисного слоя (DTO) — классы запросов/ответов для UI/API в `Services/Models`, не должны требовать знания EF и `DbContext`

### Связи и связанные поля (общие определения)

- **FK (`...Id` / `...Ids`)**: хранит “ссылку” на связанную сущность через её первичный ключ
- **Навигация**: объект/коллекция связанных сущностей; это часть объектной модели, а не обязательное наличие загруженных данных
- **Обязательная/необязательная связь**: required = non-nullable FK + non-nullable навигация; optional = nullable FK + nullable навигация
- **Загрузка связей**: наличие FK не означает, что навигации/коллекции уже загружены из БД
- **Чтение (QueryOptions + `Include*`)**: управляет тем, какие связи подгружать при чтении (например, `IncludeRelatedEntities = true`)
- **Запись (CommandOptions + `*Ids`)**: задаёт, как обновлять связи в Create/Update (какие связанные сущности “должны быть привязаны”)
- **Границы ответственности**: репозиторий загружает/применяет связи; сервис выбирает нужные опции и управляет транзакциями; контроллер вызывает только сервис

### Структура проекта

```
src/ProjectName/
├── Datasource.Ef/        # Слой данных
│   ├── Models/           ## Модели Entity Framework
│   ├── Configurations/   ## Конфигурации EF
│   ├── Contexts/         ## DbContext
│   └── Migrations/       ## Миграции БД
├── Repositories.Ef/      # Слой репозиториев
│   ├── Api/              ## Интерфейсы и реализации
│   └── Options/          ## Опции запросов
├── Services/             # Слой сервисов
│   ├── Api/              ## Интерфейсы и реализации сервисов
│   ├── Models/           ## Модели запросов/ответов
│   └── Texts/            ## Текстовые константы
└── Client/               # Слой представления
    ├── Controllers/      ## Контроллеры MVC
    └── Views/            ## Razor представления
```
